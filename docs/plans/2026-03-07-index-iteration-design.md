# 40 Below Homepage Iteration Design

## Summary

Narrow the site down to a single chosen direction based on `index.html`. Remove the unused variant files, add a simple top navigation link to support, and move the description content into the hero so the icon, name, subtitle, promotional text, App Store badge, and description all live in one top-aligned section beside the screenshot.

## Constraints

- Keep using metadata-only copy.
- Keep the official App Store badge.
- Preserve the simplified palette.
- Add a top navigation path to `support.html`.
- Keep the screenshot column aligned to the top of the hero.

## Layout

The page structure becomes:

1. Top nav with the product mark and a `Support` link
2. One hero section with:
   - icon
   - product name
   - subtitle
   - promotional text
   - App Store badge
   - description paragraphs
   - feature bullets
   - location sentence
   - screenshot in the adjacent column
3. Release notes block
4. Footer copyright

The hero content column and screenshot column both align at the top edge.
